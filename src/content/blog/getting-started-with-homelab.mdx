Are you a curious person who likes to play with new things? Do you have an old computer laying around? Congratulations! Homelabbing is right for you!

## What is a Homelab?

Put simply, a "home lab" is a group of devices in your home that can talk to each other for the purposes of experimenting and learning. [Linux Handbook](https://linuxhandbook.com/homelab/) defines the term as

> ...the name given to a server (or multiple server setup) that resides locally in your home and where you host several applications and virtualized systems for testing and developing or for home and functional usage.

Typically they include at least one "server", or computer that _serves_ some form of application or media to your network. Servers often have _services_ running on them that can provide some benefit to you. Many of the services include a web interface for interacting with them in a browser just like you would any ordinary website.

Some concepts and technologies you will inevitably become familiar with after starting your homelab journey include Docker, Linux, virtual machines, shared network storage, and networking and firewall rules.

THe one technology I will emphasize the most here and other places on my blog is Docker! I learned everything I know about Docker from following tutorials on how to run services in my homelab, and now I use it daily in my job!

## Why should you build one?

Homelabs can be one of the best places to learn new things since you have control over all aspects of it. Compared to on-the-job training for an IT role, you can choose which projects to start from scratch and don't need to stress about providing the best user experience for customers -- all while building a strong foundation of knowledge along the way. You can also build servers to any specification you have the budget for and spin up virtual machines and Docker containers to your heart's content!

Services are one of the main lures into homelabbing for me and many others. So many free and open source (FOSS) applications are available from the community for you to run locally and harness the benefits they provide you with! Here are some examples of service categories and recommendations that you can go and explore:

- Media Server ([Plex](https://www.plex.tv/), [Jellyfin](https://jellyfin.org/), [Emby](https://emby.media/))
- Home automation ([Home Assistant](https://www.home-assistant.io/))
- Organization ([Mealie](https://mealie.io/), [Paperless-ngx](https://docs.paperless-ngx.com/))
- File storage ([Nextcloud](https://nextcloud.com/), [FileBrowser](https://filebrowser.org/))
- Monitoring ([Uptime Kuma](https://uptime.kuma.pet/), [Grafana](https://grafana.com/), [Notifiarr](https://notifiarr.com/))
- Game server ([Pterodactyl](https://pterodactyl.io/), [Pelican](https://pelican.dev/))

## "But I have to use a Command Line!"

It's not as bad as you think! Most services that you will be running in your homelab will have web interfaces that you will use after the initial installation. These are essentially websites that are running on some [port](https://www.cloudflare.com/learning/network-layer/what-is-a-computer-port/) on your machine and can be accessed in a web browser at an address like `http://<ip-of-your-server>:<port-the-service-runs-on>`. For example, `http://localhost:8080` or `http://192.168.1.10:9000`.

Many people start their homelabbing journey to get more familiar with Linux and that can be done in a variety of ways. Two common scenarios in which you may encounter Linux are highlighted below.

1. Docker containers are almost always built from a tiny Linux image called Alpine, and if you ever need to poke around inside of the container while it's running (`docker exec -it <container-name> /bin/bash`), you will encounter a Linux filesystem.
2. Virtual Machines (VMs) in the homelab are commonly configured to run a Linux server distribution. You can create VMs using <abbr title="Virtualization platform">hypervisors</abbr> and can connect to them with a remote-desktop client (if you configured a desktop environment such as [Gnome](https://www.gnome.org/) or [KDE](https://kde.org/)) or login to the machine using <abbr title="Secure SHell protocol">SSH</abbr> to access the command-line interface. See my post on [Introduction to Hypervisors](./intro-to-hypervisors) to learn more about how to get started with VMs!

## Portainer

The first service I suggest installing in your homelab is Portainer. It helps you spin up containers quickly with an intuitive <abbr title="Graphical User Interface">GUI</abbr>.

I prefer using docker compose (or Portainer "Stacks").

Using Portainer abstracts away the commands you need to run in the terminal, which is helpful for following tutorials that might assume you're using a different operating system than you actually are.

I made the switch to using Ubuntu linux full-time a few years ago so that I could become more familiarized with the <abbr title="Operating System">OS</abbr>. It proved to be a great learning experience and I am now much more comfortable installing software and interacting with the filesystem using the terminal. See [this other post](./why-linux-as-primary-os) for the whole story on why this switch to Linux was so impactful to my learning.
