Are you a curious person who likes to play with new things? Do you have an old computer laying around? Congratulations! Homelabbing is right for you!

## What is a Homelab?

A "home lab" is simply some devices in your home that can talk to each other. Typically they include at least one "server", or computer that _serves_ some form of media to your network. Servers often have _services_ running on them that can provide some benefit to you. Many of the services include a web interface for interacting with them in a browser just like you would any ordinary website.

Could be setting up software services
Shared network storage
Usually involves connecting devices together on your network.
Docker is a must. Very easy to learn! See [this post](./first-docker-containers) for how to get started with Docker!

## Why you should start building a homelab

Excellent place to learn and test new things. Easily transferrable to your IT job, or can even help you get a new job.
Have extra hardware laying around? Virtually all computers can be used as servers!

## But I Have to Use a Command Line!

It's not as bad as you think! Most services that you will be running in your homelab will have web interfaces that you will use after the initial installation. These are essentially websites that are running on some port on your machine and can be accessed at `http://<ip-of-your-server>:<port-the-service-runs-on>`.

Many people start their homelabbing journey to get more familiar with Linux and that can be done in a variety of ways. First, Docker containers are almost always built on a tiny Linux image called Alpine, and if you ever need to poke around inside of the container (`docker exec -it <container-name> /bin/bash`), you will encounter a Linux filesystem.

The next scenario revolves around Virtual Machines (VMs). You can create VMs through some sort of <abbr title="Virtualization platform">hypervisor</abbr> and use a remote-desktop client to interact with it (if you configured a desktop environment such as [Gnome](https://www.gnome.org/) or [KDE](https://kde.org/)) or login to the machine with <abbr title="Secure SHell protocol">SSH</abbr> to access the command-line interface. See my other post on [Introduction to Hypervisors](./intro-to-hypervisors) to learn more.

## Portainer

The first service I suggest installing in your homelab is Portainer. It helps you spin up containers quickly with an intuitive <abbr title="Graphical User Interface">GUI</abbr>.

I prefer using docker compose (or Portainer "Stacks").

Using Portainer abstracts away the commands you need to run in the terminal, which is helpful for following tutorials that might assume you're using a different operating system than you actually are.

I made the switch to using Ubuntu linux full-time a few years ago so that I could become more familiarized with the <abbr title="Operating System">OS</abbr> It proved to be a great learning experience and I am now much more comfortable installing software and interacting with the filesystem using the terminal. See [this other post](./why-linux-as-primary-os) for the whole story on why this switch to Linux was so impactful to my learning.
